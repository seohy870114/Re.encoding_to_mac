# 📝 Project: Persona-AI_Gradio 개발 및 운영 기록

**날짜:** 2026-02-27 (Friday)
**주제:** Gradio 로컬 서버 관리 및 프로세스 문제 해결 가이드

## 1. 🛑 Gradio 서버 종료 및 관리

* **표준 종료:** 터미널 창에서 **`Control + C`**를 입력하여 실행 중인 파이썬 프로세스에 중단 신호를 보냄.
* **강제 종료:** `Control + C`가 작동하지 않거나 포트가 점유된 경우, 프로세스 ID(PID)를 찾아 강제 종료 처리함.

## 2. 🔍 PID와 Port의 개념 이해

* **Port (포트):** 주소 뒤의 숫자(예: 7860, 7861)로, 특정 서비스가 외부와 통신하기 위해 열어둔 **'논리적인 문 번호'**.
* **PID (Process ID):** 운영체제가 실행 중인 각 프로그램에 부여한 **'고유 식별 번호'**. 하나의 포트는 특정 PID를 가진 프로세스에 의해 점유됨.

## 3. 🧟 '좀비 상태' 프로세스 대응

* **발생 원인:** 터미널 탭을 그냥 닫거나 서버가 비정상 종료될 경우, 포트는 여전히 '사용 중'으로 남고 프로세스가 백그라운드에서 살아있는 상태가 됨.
* **확인 방법 (Mac):** 
```bash
lsof -i :[포트번호]
```

* **강제 종료 명령어:**
```bash
kill -9 [확인된 PID]
```



## 4. 🌐 종료 후 브라우저 연결 상태 (CLOSED/TIME_WAIT)

* **현상:** 서버 종료 후 `lsof` 명령어 실행 시 `(CLOSED)` 또는 `(TIME_WAIT)` 상태의 메시지가 출력됨.
* **원인:** 서버(Gradio)는 닫혔으나, 이전에 연결되었던 브라우저(Client)가 네트워크 통신 통로를 완전히 정리하지 못해 남겨진 잔상임.
* **해결:** 브라우저 탭을 닫거나 새로고침하면 운영체제가 해당 연결을 완전히 회수하며, 이후 `lsof` 결과에서 사라짐.
